    第九章：函数¶

    Shell 函数使用函数保留字或特殊语法 funcname() 定义。Shell 函数在内部读取和存储。读取函数时将解析别名。函数的执行与命令类似，其参数作为位置参数传递。（请参见 命令执行。）

    函数在与调用方相同的进程中执行，并与调用方共享所有文件和当前工作目录。在函数内部设置的 EXIT trap 会在函数在调用者的环境中完成后执行。

    内置的 return 用于从函数调用中返回。

    函数标识符可以被内置的 functions 列出。函数可以用内置的 unfunction 取消定义。

    自动加载的函数¶

    可以使用 autoload 内置函数将函数标记为 undefined （或 functions -u 或 typeset -fu）。这样的函数没有函数体。首次执行该函数时，Shell 会使用 fpath 中的项目搜索其定义。因此，要定义自动加载的函数，典型的顺序是：

    fpath=(~/myfuncs $fpath)
autoload myfunc1 myfunc2 ...
    如果为 autoload 内置函数或其等效项提供选项 -U，则读取过程中通常的别名扩展将被抑制。建议使用此方法时使用 Zsh 发行版提供的函数。请注意，对于使用 zcompile 内置命令预编译的函数，创建 .zwc 文件时必须提供标志 -U，因为相应的信息被编译到后者中。

    对于 fpath 中的每一项，Shell 会查找三种可能的 文件，其中最新的用于加载函数的定义：element

    element.zwc
    使用 zcompile 内置命令创建的文件，预期 包含名为 的目录中所有函数的定义。该文件的处理方式与目录相同 包含函数的文件，并搜索 功能。如果未找到定义，则搜索定义 继续进行下面描述的其他两种可能性。element
    如果已经包含 .zwc 扩展名（即扩展名 由用户明确给出），则会搜索 函数的定义，而不将其年龄与其他函数的年龄进行比较 文件;实际上，不需要任何命名的目录没有后缀。因此，包括 fpath 中的 '/usr/local/funcs.zwc' 会加快对 函数，缺点是包含的函数必须显式 在 shell 注意到任何更改之前手动重新编译。elementelementelement
    element/function.zwc
    使用 zcompile 创建的文件，该文件应包含 的定义。它可能包括其他函数定义 以及，但这些既未加载也未执行;在此中找到的文件 way 仅搜索 的定义。functionfunction
    element/function
    zsh 命令文本的文件，被视为 的定义。function

    总之，搜索顺序是，首先，在 fpath 中目录的父目录中，查找已编译目录或 fpath 中的目录;其次，如果其中多个包含 定义 对于所寻求的函数，选择 fPath 中最左边的;第三，在目录中，已编译的 function 或普通函数定义。

    如果设置了 KSH_AUTOLOAD 选项，或者文件仅包含 简单定义函数，则将执行文件的内容。 这通常会定义有问题的函数，但也可以执行 initialization，在函数执行的上下文中执行， ，因此可以定义局部参数。如果函数 不是通过加载文件来定义的。

    否则，函数体（周围没有 '（） {}'） 被视为文件的完整内容。这 form 允许将文件直接用作可执行 shell 脚本。如果 处理会导致函数被重新定义，则 函数本身不会重新执行。强制 shell 执行 initialization 然后调用定义的函数，则文件应包含 初始化代码（将执行然后丢弃）以及 完整的函数定义（将保留以供后续调用 添加到函数中），以及调用 shell 函数（包括任何 arguments。funcname...

    例如，假设自动加载文件 func 包含

    func() { print This is func; } print func is initialized 那么 'func; func' 和 KSH_AUTOLOAD 集将产生两条消息 第一次调用时，但只有消息“This is func” 在第二次调用时 和后续调用。如果没有 KSH_AUTOLOAD 设置，它将产生 第一次调用时的初始化消息，以及 第二次和后续调用。

    也可以创建一个未标记为 autoloaded 的函数。 但是它通过搜索 fpath 来加载自己的定义，通过使用 'autoload -X' 在 shell 函数中。例如，以下是 等效：

    myfunc() { autoload -X } myfunc args... 和

    unfunction myfunc # if myfunc was defined autoload myfunc myfunc args... 事实上，functions 命令将 'builtin autoload -X' 输出为 自动加载函数的主体。这样做是为了

    eval "$(functions)" 产生合理的结果。真正的自动加载函数可以是 通过正文中存在注释 '# undefined' 来标识， 因为所有注释都从定义的函数中丢弃。

    加载自动加载函数 myfunc 的定义，而不加载 执行 myfunc，使用：

    autoload +X myfunc
